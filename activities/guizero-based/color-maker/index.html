Color Maker

What will I make?

An application to create new colors by mixing red, green, and blue.

Please read the Guizero documentation about colors before continuing. This will teach you different ways to specify colors in Guizero.

The GUI is comprised of a single window () with these widgets:

  • three images () for the header
  • a slider () to enter the amount of red (red_slider)
  • a slider () to enter the amount of green (green_slider)
  • a slider () to enter the amount of blue (blue_slider)
  • a color patch () to show the resulting color (patch_text)

Instructions

  1. Download and uncompress the starter kit ZIP archive which contains all the resources (eg images) you need and some code to get you started..
  2. Open the main.py file in your favourite editor (eg IDLE)..
  3. Run the program. A window should pop up but nothing will happen when you move the sliders. This is because the code is incomplete but you will fix this next..
  4. Modify main.py, adding the missing code in the places marked with # TODO. In particular:
    • Implement the on_change_red callback function for when the red slider is moved:
      1. The new amount of red is passed to the function as a string parameter (slider_value).
      2. Convert slider_value to an integer and store it into a variable (r).
      3. Set the background color of slider_red to (r, 0, 0).
      4. Set the foreground color of slider_red to white if r < 180 or to black else.
      5. Call the mix_colors() function which will mix the 3 colors and update patch_text.
      Show me def on_change_red(slider_value): r = int(slider_value) red_slider.bg = (r, 0, 0) red_slider.text_color = "white" if r < 180 else "black" mix_colors()
    • Implement the on_change_green callback function for when the green slider is moved:
      1. The new amount of green is passed to the function as a string parameter (slider_value).
      2. Convert slider_value to an integer and store it into a variable (g).
      3. Set the background color of slider_green to (0, g, 0).
      4. Set the foreground color of slider_green to white if g < 180 or to black else.
      5. Call the mix_colors() function which will mix the 3 colors and update patch_text.
      Show me def on_change_green(slider_value): g = int(slider_value) green_slider.bg = (0, g, 0) green_slider.text_color = "white" if g < 180 else "black" mix_colors()
    • Implement the on_change_blue callback function for when the blue slider is moved:
      1. The new amount of blue is passed to the function as a string parameter (slider_value).
      2. Convert slider_value to an integer and store it into a variable (b).
      3. Set the background color of slider_blue to (0, 0, b).
      4. Set the foreground color of slider_blue to white.
      5. Call the mix_colors() function which will mix the 3 colors and update patch_text.
      Show me def on_change_blue(slider_value): b = int(slider_value) blue_slider.bg = (0, 0, b) blue_slider.text_color = "white" mix_colors()

If you would prefer writing all the Python code by yourself, download and uncompress the resources ZIP archive. It contains all the resources (eg images) but no Python code.

If you would like to see the finished app, download and uncompress the solution ZIP archive. It contains all the resources (eg images) and the full Python code.

What can I try next?

Here are some things you can try to make the app better. They are independent from each other so you can pick just a couple or do them in a different order. If you get stuck, use the Hints button or read the Guizero documentation (see Build an app with Guizero).

And of course you can also use your imagination and change the app as you like!

  • Learn more about colors with the W3Schools Colors Tutorial.
  • Add a button to pick a random color.
    1. Create a new with text="Random" (or an image) and linked to the on_random callback function.
    2. Create a new on_random callback function to pick random values between 0-255 for red/green/blue and update the display.
      3. Hints
      • You can generate a random number between 0-255 with random.randint(0, 255).
      • Setting the value of a slider, eg red_slider.value = ... will cause it's callback function to execute automatically.
      • Adding the new button will use up some vertical space and you will need to increase the height of the accordingly.
  • Add a drop-down list with predefined colors.
    1. Create a new dictionary stored in the variable colors. This dictionary contains key-value pairs where the key is the color name and the value is a tuple with the R/G/B values. The dictionary can be hard-coded (ie defined in the code) or loaded from a text file (eg see colors.csv in the resources folder).
      2. Hints
      • Here is an example of a hard-coded dictionary: colors = { "White": (255, 255, 255), "Black": (0, 0, 0), "Gray": (128, 128, 128), "Red": (255, 0, 0), "Green": (0, 255, 0), "Blue": (0, 0, 255), "Yellow": (255, 255, 0), "Cyan": (0, 255, 255), "Magenta": (255, 0, 255) }
      • The resources/colors.csv file is in comma-separated-values (CSV) format. Its content looks like this: COLOR_NAME,RED,GREEN,BLUE Black,0,0,0 White,255,255,255 Red,255,0,0 Green,0,255,0 Blue,0,0,255 ...
      • Read the File Handling section from W3Schools to find how to read a text file.
      • The color name and R/G/B values can be extracted from a line of text using the str.split() function, eg with something like name, r, g, b = line.split(",").
    2. Create a new with options=list(colors.keys()) and linked to the on_predefined callback function.
    3. Create a new on_predefined callback function to change the color.
      4. Hints
      • The callback function needs to take color_name as a parameter.
      • The RGB tuple can be retrieved from color_name with something like color_rgb = colors[color_name].
      • The amouunt of R/G/B can be retrieved from color_rgb with something like r, g, b = color_rgb.
      • Finally you need to change the value of each slider, eg red_slider.value = r.
